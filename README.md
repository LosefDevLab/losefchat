## 停止对 XSC 的开发

我们决定停止对 XShChat (XSC) 的开发，并从头开始构建一个全新的聊天模块。原因如下：
- 版本管理混乱，功能不够完善。
- 长期未更新（鸽了好几个月）。
- 用户体验不佳，许多服主选择开发第三方服务端或客户端。

尽管目前仍有许多服务器（如 `kkko`、`pppo`）在使用 XSC，并对其充满美好回忆，但我们不得不做出这一艰难决定。

在此，我们向您保证，新项目 **LosefChat** 将致力于改进这些问题，并提供更好的用户体验。



## LosefChat 与 XShChat 的对比

由于 XSC 几乎完全由“人类”生成（仅有 1% 为“AI”生成），其用户体验存在诸多问题。以下是两者的详细对比：

### 主要问题
1. **长期不更新**  
   许多服主不得不自行开发新功能。
2. **反人类设计**  
   测试服务器时，客户端和服务器是分开的。
3. **编译困难**  
   需要手动创建 `.NET` 项目并拉取代码。
4. **缺乏关键功能**  
   白名单和密码功能

### 我们的改进
- **合并服务器和客户端**：无需分别启动，部署更加便捷。
- **优化文件系统**：修复了 XSC 中懒得解决的问题。
- **增强用户体验**：重新设计回复文本，使其更加友好。
- **新增功能**：
  - 支持 IPv4 和 IPv6 协议。
  - 提供详细日志记录和搜索功能。
  - 引入 MOD AREA，支持模组开发。

## 部署极其方便

- **启动服务器**  
  输入 `2`，然后输入端口号即可启动服务器。
- **内网穿透**  
  如果您的电脑本身处于公网环境，则无需配置内网穿透。
- **命令控制**  
  简单输入一小段文本即可对服务器进行操作。
- **连接客户端**  
  输入 `1`，然后输入 IP 和端口即可连接服务器，开始聊天。
- **友好的下线方式**  
  输入 `exit`，系统会自动发送“我下线了拜拜”的消息。  
  （不过我们猜大多数人还是会直接关闭控制台窗口。）

## 模组支持

LosefChat 提供了强大的模组支持功能：
1. 拉取 LosefChat 的代码。
2. 将模组代码复制到模组开发区域。
3. 在模组运行区域添加安装代码。
4. 重新编译 LosefChat，即可使用第三方功能。

## 如何编译？

请确保已安装 `dotnet` 和 `git`（版本 >= 5.0），然后按照以下步骤操作：

```bash
git clone https://github.com/LosefDevLab/losefchat.git
cd losefchat
dotnet build
cd bin
cd Debug
# 进入对应版本目录，例如 net5.0
./losefchat
```
